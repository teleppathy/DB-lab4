# Звіт до роботи: SQL Запити (JOIN, Aggregation, Grouping)

## 1. JOIN queries (З'єднання таблиць)
Ці запити використовуються для об'єднання даних з кількох пов'язаних таблиць.
* **1. Клієнти та їхні підписки:** Показує ім'я клієнта, тип його підписки та ціну. *Чому:* Щоб бачити, якими тарифними планами користуються конкретні люди (або хто ще не має підписки завдяки LEFT JOIN).
![screenshot](img/customer%20and%20subscription.png)
* **2. Фільми та студії:** Виводить назву фільму та студію, яка його зняла. *Чому:* Щоб ідентифікувати виробника кожного контенту в каталозі.
* **3. Фільми та актори:** Показує акторський склад кожного фільму (через проміжну таблицю `film_actor`). *Чому:* Щоб користувачі могли бачити, хто грає в конкретному кіно.
* **4. Фільми та жанри:** Об'єднує фільми з їхніми жанрами. *Чому:* Для категоризації контенту та налаштування фільтрів пошуку.
* **5. Історія платежів клієнтів:** Показує суми та статуси оплат для кожного користувача. *Чому:* Для відстеження фінансових транзакцій і стану рахунку клієнта.

## 2. Base aggregation (Базова агрегація)
Запити з використанням базових математичних функцій (SUM, COUNT, AVG, MIN).
* **1. Загальна сума витрат клієнта:** Рахує, скільки грошей приніс кожен користувач. *Чому:* Щоб знайти найбільш прибуткових (VIP) клієнтів (LTV).
![screenshot](img/total%20sum%20of%20cutomer's%20payments.png)
* **2. Кількість фільмів кожної студії:** Рахує фільми по студіях. *Чому:* Для аналізу продуктивності партнерів-виробників.
* **3. Середня ціна підписки:** Визначає середній чек тарифних планів. *Чому:* Для розуміння загальної цінової політики сервісу.
![screenshot](img/avg%20price.png)
* **4. Найдешевша підписка:** Знаходить мінімальну ціну тарифу. *Чому:* Для маркетингу (наприклад, "Ціни стартують від...").
* **5. Кількість фільмів на одного актора:** Рахує, у скількох стрічках знявся кожен актор. *Чому:* Щоб виділити найбільш затребуваних акторів бази.
![screenshot](img/films%20per%20actor.png)

## 3. Grouping (Групування)
Запити, що групують дані за певними ознаками (GROUP BY).
* **1. Фільми за роком випуску:** Показує кількість релізів у кожному році. *Чому:* Для аналізу віку контенту в бібліотеці сервісу.
![screenshot](img/films%20per%20year.png)
* **2. Статистика цін за типами підписок:** Рахує мін., макс. та середню ціну для кожного типу. *Чому:* Для порівняння фінансових показників різних тарифних сіток.
* **3. Фільми за віковим обмеженням:** Групує каталог за рейтингом (16+, 18+ тощо). *Чому:* Щоб розуміти, на яку аудиторію орієнтовано більшість контенту.
* **4. Платежі за статусом:** Рахує кількість успішних та неуспішних оплат. *Чому:* Для моніторингу стабільності платіжної системи.
* **5. Кількість акторів у фільмах:** Показує розмір акторського складу кожної стрічки. *Чому:* Щоб знайти масштабні проекти з великою кількістю ролей.

## 4. Groups filtering (Фільтрація груп)
Запити з використанням `HAVING` для фільтрації вже згрупованих даних.
* **1. Клієнти з помилками оплат:** Знаходить користувачів, у яких є хоча б один неуспішний платіж. *Чому:* Для роботи служби підтримки (можливо, проблеми з карткою).
* **2. Актори з > 2 фільмами:** Відфільтровує акторів, які мають більше двох ролей. *Чому:* Щоб виділити головних зірок платформи.
* **3. Непопулярні підписки:** Знаходить тарифи, які мають менше 2 користувачів. *Чому:* Для перегляду бізнес-моделі (можливо, тариф варто скасувати).
![Непопулярні підписки](img/subscription%20type%20with%20less%20then%202%20cus.png)
* **4. Клієнти, що витратили > 100:** Залишає лише тих, хто сумарно успішно заплатив більше 100. *Чому:* Для нарахування бонусів чи переведення в преміум-сегмент.
* **5. Фільми з одним жанром:** Шукає кіно зі строго одним жанром. *Чому:* Щоб знайти "чисті" жанрові стрічки без домішок.
![screenshot](img/films%20with%20only%20one%20genre.png)

## 5. Multi-table aggregation (Багатотаблична агрегація)
Складні запити, що поєднують JOIN, групування та агрегатні функції.
* **1. Загальний дохід за типами підписок:** Рахує реальні зароблені гроші (успішні транзакції) по кожному тарифу. *Чому:* Для визначення найбільш прибуткового продукту компанії.
![screenshot](img/revenue%20for%20subscription.png)
* **2. Статистика режисерів:** Виводить кількість знятих фільмів та середній рік їх випуску для кожного режисера. *Чому:* Для аналізу активності та епохи творчості режисерів.
* **3. Популярність жанрів:** Рахує загальну кількість фільмів у кожному жанрі. *Чому:* Щоб знати, якого контенту найбільше (наприклад, драм чи комедій).
* **4. Сума втрачених коштів:** Рахує загальну суму неуспішних платежів. *Чому:* Для оцінки фінансових втрат сервісу через проблеми з оплатами.
* **5. Топ-5 акторів:** Визначає 5 акторів з найбільшою кількістю ролей. *Чому:* Для створення добірок типу "Найпопулярніші актори нашого сервісу".
